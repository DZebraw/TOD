# AGENT.md Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a concise, project-specific root `AGENT.md` that lets AI coding agents quickly understand the DawnTOD Unity package and work safely in it.

**Architecture:** Add one authoritative Markdown guide at the repository root. Organize it from project identity to architecture, editor workflows, modification rules, and verification so an agent can scan top-down and then jump directly to the relevant section.

**Tech Stack:** Markdown; Unity 2022.3; Unity Package Manager package layout; C# Runtime and Editor assemblies; URP/HDRP conditional compilation.

## Global Constraints

- The package name is `com.tencent.dawn.tod` and the Unity baseline is `2022.3`.
- Runtime code belongs under `Runtime`; Unity Editor-only code belongs under `Editor`.
- Preserve Unity `.meta` files and GUID relationships when handling assets.
- Treat `USING_URP` and `USING_HDRP` as render-pipeline-dependent symbols maintained by `Editor/AutoDefineRenderPipelineSymbols.cs`.
- The repository currently has no automated test files; verification must include `git diff --check` and Unity 2022.3 manual smoke checks when a host project is available.
- Do not modify existing C#, Shader, Compute Shader, Unity asset, or package metadata files for this documentation task.
