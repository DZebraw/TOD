## 阶段 1：太阳方向光程修正

### 目标

让云内太阳光自阴影由真实采样距离决定，避免修改光照步数或云层路径长度时阴影强度无规律变化。

### 实施

- 光学厚度由无单位的密度求和改为 `density * lightStepSize`。
- 未修改密度造型、相位、环境补光、动画或重建。

### 验收结果

- 2026-07-21：用户确认通过。

### 回退范围

- `Runtime/URP/DawnVolumetricCloud.shader` 中 `DawnCloudLightMarch` 的光学厚度累计。

## 阶段 2：相位响应合法化与稳定化

### 根因

- 旧默认参数的 Y 分量为 `-1.37`，Shader 又对它取反，最终向 HG 相位函数传入 `g = 1.37`。
- HG 相位函数要求 `|g| < 1`；非法值会产生负分子和不可预测的顺光、侧光、逆光响应。
- 原面板把四个含义不同的控制打包为一个 Vector4，不利于美术验收和复现。

### 实施

- 前向各向异性限制为 `0.0–0.9`。
- 后向各向异性限制为 `-0.75–0.0`，不再进行错误的符号反转。
- Z 分量明确为前向/后向散射平衡，W 分量明确为方向光响应强度。
- Renderer 和 Shader 双重限制参数范围，旧 Volume Profile 中的越界值也不会进入 HG 计算。
- Inspector 将 Vector4 显示为四个具名滑杆；本阶段不加入多重散射或环境补光。

现有 Volume Profile 会保留已经序列化的旧数值，但运行时会使用安全范围。为了统一验收基线，建议手动设置为：前向散射 `0.7`、后向散射 `-0.2`、前向/后向平衡 `0.7`、方向光响应 `1.0`、非迎光面填充 `0.18`。

### 验收反馈与修订

- 首次验收反馈：不向光的区域过黑。
- 数值复查：新相位在侧光方向约为 `0.037`，只剩旧效果常量亮度底座的约 7%；当前阶段尚未加入天空环境光，因此暗面亮度损失被直接暴露。
- 修订方式：增加独立的“非迎光面填充”。Shader 使用 `max(最低填充, 方向相位)`，只托起过暗方向，不继续抬高已经明亮的迎光高光。
- 默认值为 `0.18`。这是阶段 2 的临时可控美术底座；后续天空补光和多重散射完成后，需要重新评估并可能降低该值。

### 建议验收画面

1. 固定同一片云，分别观察顺光、侧光和逆光。
2. 旋转太阳时，云亮度应连续变化，不应突然翻黑、爆亮或出现负亮度观感。
3. 逆光可以出现方向性亮边，但不应覆盖整片云体。
4. 云的密度、轮廓、运动速度和几何交界应与阶段 1 保持一致。

### 回退范围

- `Runtime/URP/DawnVolumetricCloud.shader` 的 `DawnCloudPhase`。
- `Runtime/URP/DawnVolumetricCloudRendererFeature.cs` 的相位参数校验。
- `Runtime/AP/Scripts/DawnVolumetricCloudVolume.cs` 的相位默认值。
- `Editor/DawnVolumetricCloudVolumeEditor.cs` 的相位滑杆界面。

### 验收结果

- 2026-07-21：根据“不向光的地方太黑”反馈加入非迎光面填充后，用户确认继续下一阶段。

## 阶段 3：两阶多重散射近似

### 目标

让经过云内多次散射的太阳光能够进入厚云内部，减少单次散射造成的死黑和灰蒙，同时保留直接光形成的体积阴影与逆光方向感。

### 实施

- 参照 GLSL 中逐阶降低消光、散射贡献并让高阶相位趋向均匀分布的方案，实现直接光与第二阶散射光。
- 第二阶与直接光共用 `DawnCloudLightMarch` 已有的 8 次密度采样，不增加 3D 噪声采样次数；只新增第二组透射率、颜色和相位计算。
- 第二阶使用较低消光，让光能比直接光更深入厚云；相位在均匀相位与当前方向相位之间混合，避免内部补光完全贴着太阳方向。
- 视线方向的密度、消光与透射率保持不变，因此本阶段不会改变云的轮廓、覆盖率和整体透明度。
- `Internal Light Contribution` 设为 `0` 可恢复阶段 2 的单次散射结果，便于直接 A/B 和回退。

### 默认验收参数

- `Internal Light Extinction = 0.5`：第二阶保留一半消光；数值越低，光进入厚云越深。
- `Internal Light Contribution = 0.35`：第二阶光能强度。
- `Internal Light Directionality = 0.35`：保留少量太阳方向感，大部分趋向均匀散射。

### 建议验收画面

1. 选择具有明显厚薄变化的云团，在固定太阳与相机下，将 `Internal Light Contribution` 在 `0` 和 `0.35` 之间切换。
2. 开启后，厚云内部应出现更连续的透光层次，原本较死的暗部应被回填；薄边和云外轮廓不应明显变厚。
3. 云体仍应保留亮面、过渡面和阴影面的区分，不应整体发白或像自发光。
4. 侧光和逆光观察时，内部补光应比直接光柔和，太阳方向的亮边仍由直接光主导。
5. 密度形状、运动、场景遮挡与低分辨率重建效果应与阶段 2 一致。

### 调整与回退

- 如果画面偏奶白，先降低 `Internal Light Contribution`；如果只有最深处过亮，则提高 `Internal Light Extinction`。
- 如果补光仍过度集中在太阳方向，降低 `Internal Light Directionality`。
- 完整回退范围包括 `DawnVolumetricCloud.shader` 的第二阶光照结果、Renderer 的 `_DawnCloudMultiScatterParameters` 绑定、Volume 的三个多重散射参数，以及对应 Inspector 字段。

### 验收结果

- 2026-07-21：用户确认继续下一阶段。

## 阶段 4：天空补光与地面反照

### 目标

把非迎光面从固定亮度底座扩展为与场景天空环境色和云层上下位置相关的光照：上部主要接受天空补光，云底接受少量带地表色彩的反照，从而建立“亮边偏太阳色、阴影偏天空色、云底带地表色”的基础关系。

### 实施

- Renderer 每帧读取 `RenderSettings.ambientLight`。Dawn TOD 已有的天空环境光更新会写入该值，因此云的环境补光会跟随当前时间和天空光色，而不是使用固定白光。
- `Sky Fill Tint` 和 `Ground Bounce Tint` 分别对场景环境色做乘色；两个 Intensity 独立控制能量，便于美术分别调整冷色天空填充与暖色地面反照。
- Shader 根据采样点在云层包围盒中的相对高度计算平滑上下权重：天空光在云层顶部最强、底部仍保留 35%；地面反照在底部最强并向顶部衰减到 0。
- 环境光不经过太阳相位函数，因此旋转太阳时暗面不会随直接光一起翻黑；太阳直射、多重散射和环境补光保持清晰分工。
- 本阶段没有增加密度或噪声采样，也没有修改云形、覆盖率、运动和视线透射率。

### 默认验收参数

- `Sky Fill Tint = (0.8, 0.9, 1.0)`：略偏冷，乘以当前场景环境色。
- `Sky Fill Intensity = 0.35`：提供可读暗面，但不替代太阳直射。
- `Ground Bounce Tint = (0.65, 0.5, 0.35)`：代表平均暖色地表反照。
- `Ground Bounce Intensity = 0.12`：只轻微影响云底。

### 建议验收画面

1. 固定同一片厚云，将 `Sky Fill Intensity` 与 `Ground Bounce Intensity` 同时设为 `0`，再恢复默认值进行 A/B。
2. 开启后，侧面和背光面应获得与当前天空一致的颜色与亮度，仍能看出太阳方向；不应像整体加了一层白色自发光。
3. 从低机位观察云底：底部应比阶段 3 更可读，并有轻微地表色倾向，但不应比云顶更亮。
4. 改变 Dawn TOD 时间或场景 `Ambient Light`，云暗面的光色应同步变化；正午偏清透，日出日落可出现暖亮边与相对冷的阴影。
5. 云形、轮廓、透明度、运动和采样噪点应与阶段 3 保持一致。

### 调整与回退

- 如果整体偏亮或发灰，先降低 `Sky Fill Intensity`，不要先压暗直射光。
- 如果云底过暖或像被点亮，降低 `Ground Bounce Intensity`，其次将 `Ground Bounce Tint` 调向中性灰。
- 将两个环境光 Intensity 设为 `0`，可恢复阶段 3 的画面结果。
- 完整回退范围包括 Shader 的 `DawnCloudEnvironmentLight`、Renderer 的场景环境色解析与两个颜色绑定、Volume 的四个环境光参数，以及对应 Inspector 字段。

### 验收结果

- 2026-07-21：用户确认继续下一阶段。

## 阶段 5A：独立 Coverage 水平覆盖率

### 目标

提供一个语义稳定的云量控制：它只决定水平方向上有多少区域存在云，不再要求美术同时调整 `Density Bias`、`Density Multiplier` 和侵蚀参数来间接改变云量。

### 实施

- 复用现有 `Sky_NoiseSmooth` 低频灰度场作为水平覆盖分布，不增加纹理采样。
- `Cloud Coverage` 将灰度场转换为覆盖阈值，并使用固定 `0.12` 的软过渡形成自然云团边界，避免硬切二值轮廓。
- 覆盖遮罩作用于完整的基础密度（包括 Density Bias），因此 `Coverage = 0` 时不会被正向 Density Bias 重新生成云。
- 完全覆盖的区域保持阶段 4 的原始密度；Coverage 只影响被裁掉的区域和软边界，不作为全局密度乘数使用。
- 视线步进与太阳方向光照步进共用同一密度函数，因此覆盖变化会同步影响可见云体及其云内自阴影。
- 本阶段没有调整 Weather Map 的高度含义、云型高度曲线、细节侵蚀、光照或动画。

### 默认验收参数

- `Cloud Coverage = 0.65`：形成有间隙的常规云层。
- `Cloud Coverage = 0`：晴空，无体积云密度。
- `Cloud Coverage = 1`：绕过覆盖遮罩，恢复阶段 4 的完整云层结果。

### 建议验收画面

1. 固定相机和天气资源，依次观察 `Coverage = 0.25 / 0.65 / 0.9 / 1.0`。
2. 数值增加时，云团应主要在水平方向连接、扩张并填满空隙，不应只是原有云体整体变浓、变白。
3. 保留下来的云团主体密度和内部明暗应基本稳定；变化应集中在云团的出现、消失和软边界。
4. `Coverage = 0` 应完全无云；`Coverage = 1` 应与阶段 4 一致，便于 A/B 与回退。
5. Coverage 动态变化时不应出现硬边跳变、负密度或额外采样噪点。

### 调整与回退

- 默认值过稀时可先提高到 `0.75`，不需要同时提高 Density Multiplier。
- 本阶段的快速回退是将 `Coverage` 设为 `1`。
- 完整回退范围包括 Shader 的低频覆盖遮罩、Renderer 的 `_DawnCloudCoverage` 绑定、Volume 的 Coverage 参数及 Inspector 字段。

### 验收结果

- 2026-07-21：用户确认继续下一阶段。

## 阶段 5B：云底、主体与顶部高度塑形

### 目标

用一条含义明确的垂直密度曲线替代单纯的软片状高度衰减：云底保持相对平整且边缘柔和，中部保留饱满密度，顶部根据 Weather Map 产生不同程度的向上生长与软化轮廓。

### 实施

- 保留阶段 5A 的旧高度曲线，并以 `Height Profile Blend` 在旧曲线和新曲线间插值；设为 `0` 可精确恢复阶段 5A，设为 `1` 使用新曲线。
- 新曲线的云底只由统一高度与 `Cloud Base Softness` 决定，不再跟随 Weather Map 上下漂移，因此低机位观察时云底更连贯。
- `Cloud Body Height` 定义所有云团都能保留的主体高度；主体区域的高度权重保持为 1，避免整层云从底到顶都呈均匀软灰。
- `Vertical Growth` 使用现有 Weather Map 的低频值把选定区域从主体高度推向云层顶部：`0` 接近层云，`1` 允许最亮区域长到包围盒顶部。
- `Cloud Top Softness` 控制顶部衰减距离，使高处轮廓可以在厚实与柔软之间调整。
- 新旧曲线共用已有 Weather Map 采样，没有增加纹理或噪声采样；Coverage、光照、动画和细节侵蚀保持不变。
- 旧 `Height Weights` 仍在运行时保留，只用于 `Height Profile Blend = 0` 时的精确回退，不再作为日常美术参数显示。

### 默认验收参数

- `Height Profile Blend = 1.0`：完整启用新高度轮廓。
- `Cloud Base Softness = 0.08`：保留约 8% 云层高度的柔和底边。
- `Cloud Body Height = 0.55`：基础云体占云层高度的 55%。
- `Vertical Growth = 0.65`：Weather Map 高值区域可形成明显但不过度顶满的上升体块。
- `Cloud Top Softness = 0.20`：顶部在约 20% 云层高度内逐渐衰减。

### 建议验收画面

1. 保持 `Coverage = 0.65`，在固定相机下切换 `Height Profile Blend = 0 / 1`，确认变化只发生在垂直轮廓而非水平云量。
2. 低机位观察云底：新曲线应形成较稳定的水平底边，但边缘仍然柔和，不应出现硬切平面。
3. 侧面观察主体：中部应比旧曲线饱满，顶部应出现由 Weather Map 控制的高低起伏，不再整层同高或均匀发虚。
4. 将 `Vertical Growth` 在 `0 / 0.65 / 1` 间切换：应从低矮层状逐步过渡到更明显的积云塔状顶部，底部位置保持稳定。
5. 增减 `Cloud Top Softness` 时只应改变顶部过渡厚度，不应改变 Coverage 或云底位置。
6. 太阳光、自阴影、环境补光、运动速度和低分辨率噪点应延续阶段 5A。

### 调整与回退

- 如果云体像厚重平板，降低 `Cloud Body Height` 或提高 `Vertical Growth`。
- 如果顶部太尖碎，提高 `Cloud Top Softness`；如果顶部过于软塌，降低该值。
- 如果云底像硬平面，提高 `Cloud Base Softness`；如果云底过散，降低该值。
- 将 `Height Profile Blend` 设为 `0`，可完整恢复阶段 5A 的高度密度结果。
- 完整回退范围包括 Shader 的新高度曲线、Renderer 的两个高度参数绑定、Volume 的五个高度塑形参数及对应 Inspector 字段。
